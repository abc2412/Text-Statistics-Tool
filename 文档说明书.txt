二、文档说明书
1. 简介
文本统计工具（mywc） 是一个跨平台的命令行工具，用于统计文本文件的行数、单词数和字符数。它模仿 Linux wc 命令的核心功能，并增加了对中文文本的良好支持。

2. 功能特性
统计单个或多个文本文件

支持从标准输入（管道）读取数据

可选择显示行数（-l）、单词数（-w）、字符数（-c）

默认同时显示三项，与 wc 行为一致

多文件时自动输出总计

自动处理 UTF-8 和 GBK 编码文件

友好的错误提示（文件不存在、编码错误等）

3. 环境要求
Python 3.6 或更高版本（仅使用标准库，无需安装额外包）

支持 Windows / macOS / Linux

4. 安装与使用
4.1 获取脚本
将上述代码保存为 wc.py（或任意名称，如 mywc.py）。

4.2 基本用法
# 统计单个文件
python wc.py test.txt

# 只显示行数
python wc.py -l test.txt

# 只显示单词数和字符数
python wc.py -w -c test.txt

# 统计多个文件
python wc.py file1.txt file2.txt file3.txt

# 从管道读取
cat test.txt | python wc.py

# 查看帮助
python wc.py --help

4.3 输出格式示例

$ python wc.py example.txt
     150     1250     7800 example.txt
输出依次为：行数、单词数、字符数、文件名。

多个文件时：
$ python wc.py a.txt b.txt
      10       50      300 a.txt
      20      100      600 b.txt
      30      150      900 总计
5. 命令行参数说明
参数	长参数	说明
-l	--lines	只显示行数
-w	--words	只显示单词数
-c	--chars	只显示字符数
-L	--max-line-length	（扩展）显示最长行的长度
-h	--help	显示帮助信息
files	位置参数	要统计的文件路径，可多个；不提供则从标准输入读取
注意：如果未指定 -l、-w、-c 中的任何一个，默认显示全部三项。

6. 实际应用场景
日志分析：快速统计错误日志的行数

python wc.py -l /var/log/nginx/error.log

代码度量：统计 Python 项目的总代码行数
python wc.py -l *.py

验证数据完整性：比较两个文件的字符数是否一致

python wc.py -c original.csv > count1.txt
python wc.py -c copy.csv > count2.txt
diff count1.txt count2.txt

管道配合 grep：统计包含特定关键字的行数
grep "ERROR" app.log | python wc.py -l
7. 常见问题（FAQ）
Q1：单词是如何定义的？
A：按空白字符（空格、换行、制表符等）分割，连续空白被视为一个分隔符。中文汉字之间的分割也基于空格，如果没有空格，整个中文段落会被计为一个单词。这与 wc 行为一致。

Q2：支持哪些文件编码？
A：默认使用 UTF-8 读取，如果失败则尝试 GBK（常见于 Windows 中文环境）。如需其他编码，可以修改代码中的 encoding 参数。

Q3：大文件会不会很慢？
A：脚本将整个文件读入内存，适合几十 MB 以内的文件。对于超大文件（>500 MB），建议使用流式处理（一行一行读），但本工具为了代码简洁未实现。实际面试中说明即可。

Q4：如何让脚本像系统命令一样运行？

Linux/macOS：给脚本加执行权限 chmod +x wc.py，然后移动到 ~/bin/ 或 /usr/local/bin/，改名为 wc（注意不要覆盖系统命令）。

Windows：将脚本所在目录添加到 PATH，然后创建 wc.bat 文件：@python C:\path\to\wc.py %*。
